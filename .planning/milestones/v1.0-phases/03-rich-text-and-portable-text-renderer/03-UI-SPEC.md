## Spacing Scale

Standard 8-point scale. All values are multiples of 4. The renderer does NOT introduce its own spacing tokens — prose defaults govern internal paragraph/heading rhythm, with brand overrides applied per element.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing |
| md | 16px | Default element spacing, paragraph bottom margin |
| lg | 24px | Section padding, list item spacing |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- `prose` classes govern internal PT block spacing (`mb-4` on `<p>`, heading margins). Do not override prose spacing in the renderer component map — only override color, font, and decoration.
- Inline link underline offset: `underline-offset-4` (4px, on-scale).

Source: existing stub uses `mb-4`, existing layout components use `p-6`, `py-8`, `px-10` — all on-scale.

## Typography

The PT schema allows `normal`, `h2`, and `h3` block styles. Each must be styled distinctly.

| Role | Size | Weight | Line Height | Font | Class |
|------|------|--------|-------------|------|-------|
| Body (`normal`) | 18px (1.125rem / `text-base`) | 400 (normal) | 1.5 | Montserrat (`font-sans`) | `text-base font-normal leading-relaxed` |
| H2 | 26px (1.625rem / `text-2xl`) | 700 (bold) | 1.2 | Cardo (`font-serif`) | `text-2xl font-bold font-serif text-brand-red` |
| H3 | 20px (1.25rem / `text-lg`) | 700 (bold) | 1.3 | Cardo (`font-serif`) | `text-lg font-bold font-serif text-brand-red` |
| Label (list item) | 16px (1rem / `text-sm` or `text-base`) | 400 (normal) | 1.5 | Montserrat (`font-sans`) | `text-base font-normal` |

Weights: exactly 2 — 400 (`font-normal`) and 700 (`font-bold`).

H2 and H3 are differentiated by size (26px vs 20px), not weight. Both use `font-bold`. This aligns with the existing `.section-heading-red` pattern in `globals.css` which uses `font-extrabold`.

Inline marks:
- `strong` → `font-bold` (weight 700), no color change
- `em` → `italic`, no color change

Source: `globals.css` section-heading classes use `text-2xl md:text-3xl font-extrabold font-serif text-brand-red`. The renderer targets article-context headings (not page-level) so one step down: `text-2xl bold` for h2, `text-lg bold` for h3. Body at `text-base` matches existing stub and prose defaults.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `oklch(1 0 0)` — white | Page background, card backgrounds (`bg-white`) |
| Secondary (30%) | `--brand-pink oklch(96.485% 0.01293 352.656)` | Section container backgrounds (`section-container-pink`, form inputs) |
| Accent (10%) | `--brand-red oklch(54.739% 0.21883 14.447)` | H2 and H3 heading text in PT renderer; inline links; list marker color |
| Destructive | `--destructive oklch(0.577 0.245 27.325)` | Not used in this phase — renderer has no destructive actions |

Accent reserved for:
1. H2 and H3 text color in `PortableTextRenderer` component map
2. Inline link text color (`text-brand-red`)
3. Inline link hover underline decoration
4. Bullet/number list marker color (`marker:text-brand-red`)

Link interaction states:
- Default: `text-brand-red underline underline-offset-4`
- Hover: `hover:text-brand-dark-red hover:decoration-brand-dark-red`
- Focus-visible: `focus-visible:outline-2 focus-visible:outline-offset-2 focus-visible:outline-brand-red` (inherits global focus rule from `globals.css` layer base)

Source: `globals.css` `:root` variables + `.section-heading-red`, `.card-heading-red`, focus-visible rule on `a:focus-visible`.

## Context Wrappers

The renderer is universal — it outputs the same markup everywhere. The three parent layout components supply context-specific outer wrappers. These are NOT part of the renderer itself.

| Context | Parent Wrapper | Wrapper Classes |
|---------|---------------|-----------------|
| `InfoSection` | `<div class="section-richtext">` | `rounded-lg border max-w-4xl mx-auto bg-white border-gray-700 my-4 p-6 shadow-sm` |
| `ImageWithText` | `<div class="prose prose-lg text-black">` | `prose prose-lg text-black` (orientation-conditional centering in parent) |
| `CtaFeaturesSection` (feature cards) | `<div class="flex my-4 font-medium">` | `flex my-4 font-medium` |

The renderer must NOT apply its own max-width, margin, or padding — those come from the parent wrapper. Render output should be a fragment or a bare `<div>` with prose override classes only.

Source: `info-section.tsx` line 17, `image-with-text.tsx` line 17, `cta-features-section.tsx` line 77 — 03-CONTEXT.md D-06.

## Component Map Contract

Full specification of what `PortableTextRenderer`'s `@portabletext/react` `components` prop must render:

### Block styles

| Style | Element | Classes |
|-------|---------|---------|
| `normal` | `<p>` | `mb-4 text-base font-normal leading-relaxed` |
| `h2` | `<h2>` | `text-2xl font-bold font-serif text-brand-red mb-3 mt-6` |
| `h3` | `<h3>` | `text-lg font-bold font-serif text-brand-red mb-2 mt-4` |

### Marks (inline)

| Mark type | Element | Classes |
|-----------|---------|---------|
| `strong` | `<strong>` | `font-bold` |
| `em` | `<em>` | `italic` |
| `link` (annotation) | `<a>` | `text-brand-red underline underline-offset-4 hover:text-brand-dark-red focus-visible:outline-brand-red` |

Link `<a>` attributes:
- When `blank === true`: `target="_blank" rel="noopener noreferrer"`
- When `blank !== true`: no target, no rel

### Lists

| List type | Outer element | Item element | Classes |
|-----------|--------------|--------------|---------|
| `bullet` | `<ul>` | `<li>` | `ul: list-disc list-outside pl-6 mb-4 space-y-1` / `li: marker:text-brand-red` |
| `number` | `<ol>` | `<li>` | `ol: list-decimal list-outside pl-6 mb-4 space-y-1` / `li: marker:text-brand-red` |

### Edge cases

| Case | Behavior |
|------|----------|
| `content` is `undefined` | Return `null` |
| `content` is empty array (`[]`) | Return `null` |
| Block with no children | Skip block silently |
| Unknown `style` value | Fall through to `<p>` with `normal` classes |
| Unknown mark type | Render children as plain text (no wrapper) |

Source: 03-CONTEXT.md D-04, D-05 — "prose handles spacing/defaults, component map handles brand look."

## Copywriting Contract

This phase builds a renderer component — it is infrastructure, not a user-facing interaction flow. There are no form submissions, no CTAs introduced, and no destructive actions. Copy contracts apply only to states the renderer itself can emit.

| Element | Copy |
|---------|------|
| Primary CTA | Not applicable — renderer has no CTA |
| Empty state | None rendered — parent receives `null`, parent is responsible for any empty state |
| Error state | None rendered — renderer returns `null` on invalid input silently; no error message displayed to end user |
| Destructive confirmation | None — no destructive actions in this phase |

Source: REQUIREMENTS.md RT-01 — "Portable Text renderer ... component map matching existing visual output." No new user-facing strings introduced.
