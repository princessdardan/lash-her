# Phase 5 ‚Äî UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Phase Classification

**Type:** Infrastructure ‚Äî no new user-facing UI surfaces.

This phase wires Sanity webhooks to Next.js on-demand ISR revalidation. It adds one server-side Route Handler (`/api/revalidate/route.ts`), adds `{ next: { tags: [...] } }` options to every `client.fetch()` call in `loaders.ts`, and removes two `unstable_cache` wrappers from `layout.tsx`. No layout components, page components, or visual markup are changed.

The visual contract is a **preservation contract**: zero change to any rendered output. End users will see faster content propagation (seconds instead of 30 minutes) but no visual difference whatsoever.
