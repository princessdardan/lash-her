## Interaction State Contract

This section is unique to Phase 4. Both forms share identical state machine behavior.

### Submit Button States

| State | Visual | Copy | Class |
|-------|--------|------|-------|
| Idle (general inquiry) | Full opacity, clickable | "Send Inquiry" | `.btn-primary-red` |
| Idle (training contact) | Full opacity, clickable | "Send Application" | `.btn-primary-red` |
| Submitting | `disabled` attribute + `opacity-50` (shadcn default) | "Submitting..." | `.btn-primary-red` + `disabled` |
| Post-success | Returns to idle (form reset) | Reverts to idle label | `.btn-primary-red` |
| Post-error | Returns to idle (form NOT reset) | Reverts to idle label | `.btn-primary-red` |

### Status Banner

Rendered inside `<div aria-live="polite" role="status">` — already in both components.

| Outcome | Background | Text | Copy |
|---------|------------|------|------|
| Success | `bg-green-50` | `text-green-800` | "Thank you! Your submission has been received." |
| Error (generic) | `bg-red-50` | `text-red-800` | "Something went wrong, please try again." |
| Error (validation) | No banner | n/a — shown inline per field | See Field Error below |

### Field Error States

Triggered on blur (first pass) then on every change (subsequent). Sourced from `FieldError` component.

| Trigger | Behavior |
|---------|----------|
| Blur on required field (empty) | Show FieldError below input immediately |
| Change after touch (correcting error) | FieldError clears in real time |
| Submit with invalid fields | All required fields validated; all errors shown; banner not shown (inline errors sufficient) |
| Submit with network error | All inline errors cleared; generic error banner shown |

### Loading / Async Feedback

- `isSubmitting = true` disables the Submit button with shadcn's built-in `disabled` styling
- No spinner or skeleton — the button copy change ("Submitting...") is the only loading indicator
- Page does not navigate or scroll on submit — feedback is inline within the form card
