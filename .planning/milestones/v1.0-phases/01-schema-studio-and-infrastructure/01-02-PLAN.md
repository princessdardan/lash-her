IMPORTANT: The `menuDropdownSection` type inlines what was previously the `section` collection type in Strapi. The Strapi `section` had `heading` (string) and `links` (repeatable `menu.menu-link` component). This maps to `heading` (string) and `links` (array of `menuLink` objects). However, note that Strapi's `menu.menu-link` (used inside `section.links`) is DIFFERENT from `components.menu-link`. The `menu.menu-link` has `title` and `url` only. But the `section.links` component reference points to `menu.menu-link` which has just `title` and `url`. So the `links` field in `menuDropdownSection` should use `menuLink` objects (from components.menu-link with name, url, description, icon) since that's the richer type used in the actual frontend navigation dropdown rendering. This matches how the frontend currently renders dropdown sections.

```typescript
defineField({
  name: "perks", // or "info" or "features"
  title: "Perks", // or "Info" or "Features"
  type: "array",
  of: [
    {
      type: "block",
      styles: [
        { title: "Normal", value: "normal" },
        { title: "H2", value: "h2" },
        { title: "H3", value: "h3" },
      ],
      lists: [
        { title: "Bullet", value: "bullet" },
        { title: "Numbered", value: "number" },
      ],
      marks: {
        decorators: [
          { title: "Strong", value: "strong" },
          { title: "Emphasis", value: "em" },
        ],
        annotations: [
          {
            name: "link",
            type: "object",
            title: "Link",
            fields: [
              { name: "href", type: "string", title: "URL" },
              { name: "blank", type: "boolean", title: "Open in new tab" },
            ],
          },
        ],
      },
    },
  ],
})
```

IMPORTANT PATTERN for image fields with alt text:
```typescript
defineField({
  name: "image",
  title: "Image",
  type: "image",
  options: { hotspot: true },

      name: "alt",
      title: "Alternative Text",
      type: "string",
    }),
  ],
})
```
