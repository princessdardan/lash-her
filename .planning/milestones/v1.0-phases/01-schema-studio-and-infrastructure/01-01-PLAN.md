<task type="auto">
  <name>Task 2: Create read client and write client modules</name>
  <files>frontend/src/sanity/lib/client.ts, frontend/src/sanity/lib/write-client.ts</files>
  <read_first>
    - frontend/src/sanity/env.ts (just created — confirm exports)

  <action>
1. Create `frontend/src/sanity/lib/client.ts`:
   ```typescript
   import { createClient } from "next-sanity";
   import { apiVersion, dataset, projectId } from "../env";

   export const client = createClient({
     projectId,
     dataset,
     apiVersion,
     useCdn: true,
   });
   ```
   Use `createClient` from `next-sanity` (not `@sanity/client`) — the `next-sanity` version adds Next.js cache tag integration needed for Phase 5 revalidation.

2. Create `frontend/src/sanity/lib/write-client.ts`:
   ```typescript
   import "server-only";
   import { createClient } from "@sanity/client";
   import { apiVersion, dataset, projectId } from "../env";

   export const writeClient = createClient({
     projectId,
     dataset,
     apiVersion,
     useCdn: false,
     token: process.env.SANITY_WRITE_TOKEN,
   });
   ```
   Key differences from read client:
   - `import "server-only"` as FIRST import — causes build error if imported in client component
   - Uses `createClient` from `@sanity/client` (not `next-sanity`) — write client doesn't need Next.js cache integration
   - `useCdn: false` — CDN is read-only, writes must go direct
   - `token: process.env.SANITY_WRITE_TOKEN` — no NEXT_PUBLIC_ prefix, server-only secret
  </action>
