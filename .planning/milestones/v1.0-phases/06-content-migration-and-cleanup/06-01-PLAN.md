2. Create `frontend/scripts/migrate-strapi-to-sanity.ts` with the following structure:

**Imports and client setup:**
```typescript
import { createClient } from '@sanity/client'
import { nanoid } from 'nanoid'
import { Readable } from 'node:stream'
import * as dotenv from 'dotenv'

dotenv.config({ path: '.env.local' })
```

**Environment validation:** Check that all 5 required env vars exist (`NEXT_PUBLIC_SANITY_PROJECT_ID`, `NEXT_PUBLIC_SANITY_DATASET`, `SANITY_WRITE_TOKEN`, `STRAPI_BASE_URL`, `STRAPI_API_TOKEN`). If any are missing, log `[migrate] ERROR: Missing required env vars: ${missing.join(', ')}` and `process.exit(1)`.

**Sanity client:** Create with `createClient({ projectId, dataset, apiVersion: '2026-03-24', token, useCdn: false })`. Do NOT import from `write-client.ts` (it has `import "server-only"` which fails outside Next.js).

**`fetchStrapi<T>(path: string): Promise<T>` function:**
- Fetch from `${STRAPI_BASE_URL}${path}` with `Authorization: Bearer ${STRAPI_API_TOKEN}` header
- On non-200, throw with `[migrate] ERROR fetching ${path}: ${res.status}`
- Return `json.data ?? json` (Strapi 5 flat format)

**`fetchStrapiPaginated<T>(path: string): Promise<T[]>` function:**
- For collection types — fetches pages of 100 until all items returned
- Uses `?populate=deep&pagination[pageSize]=100&pagination[page]=${page}`
- Returns concatenated array of all items

**Image upload system:**
- `imageCache: Map<string, string>` — maps Strapi image URL to Sanity asset `_id`
- `uploadImage(url: string, strapiId: string | number): Promise<string>` — checks cache first, then fetches image and uploads via `client.assets.upload('image', Readable.fromWeb(res.body!), { filename: url.split('/').pop(), source: { name: 'strapi', id: String(strapiId), url } })`. Returns `asset._id`. Logs `[migrate] Uploading image: ${filename}` for each upload. On error, logs `[migrate] ERROR uploading image ${filename}: ${error.message} -- skipping` and returns empty string.
- `imageRef(assetId: string)` — returns `{ _type: 'image', asset: { _type: 'reference', _ref: assetId } }`
- `fileRef(assetId: string)` — returns `{ _type: 'file', asset: { _type: 'reference', _ref: assetId } }` (for video)

**Rich text converter `convertBlocksToPortableText(nodes: StrapiBlockNode[]): object[]`:**
- Define `StrapiTextChild` interface: `{ type: 'text', text: string, bold?: boolean, italic?: boolean, underline?: boolean, strikethrough?: boolean, code?: boolean }`
- Define `StrapiBlockNode` interface: `{ type: 'paragraph' | 'heading' | 'list' | 'list-item' | 'link' | 'quote' | 'code' | 'image', level?: number, format?: 'ordered' | 'unordered', url?: string, text?: string, children?: Array<StrapiTextChild | StrapiBlockNode>, image?: { url: string, alternativeText?: string } }`
- `paragraph` / `quote` -> `{ _type: 'block', _key: nanoid(12), style: 'normal', children: [...spans], markDefs: [] }`
- `heading` -> style is `node.level === 2 ? 'h2' : 'h3'` (map h1->h2, anything else->h3 per Phase 1 D-13)
- `list` -> for each child list-item, create `{ _type: 'block', _key: nanoid(12), style: 'normal', listItem: node.format === 'ordered' ? 'number' : 'bullet', level: 1, children: [...spans], markDefs: [] }`
- `flattenChildren(children)` -> for each text child: `{ _type: 'span', _key: nanoid(12), text: child.text, marks: [...(child.bold ? ['strong'] : []), ...(child.italic ? ['em'] : [])] }`
- For `link` children inside text: create a markDef `{ _type: 'link', _key: linkKey, href: child.url }` and add `linkKey` to the span's marks array
- If nodes is null/undefined/empty, return empty array

**Block transformer `transformBlock(block: any): Promise<object | null>`:**
- Map Strapi `__component` string to Sanity `_type`: strip `layout.` prefix, convert kebab-case to camelCase (e.g., `layout.hero-section` -> `heroSection`, `layout.cta-features-section` -> `ctaFeaturesSection`)
- For each block type, map Strapi fields to Sanity fields based on the Sanity schema
- Handle image fields by looking up `imageCache` or uploading
- Handle rich text fields (`info`, `perks`, `features`) by calling `convertBlocksToPortableText()`
- Add `_key: nanoid(12)` to each block
- Special handling for `photoGallery`: Strapi field `image` (multiple) -> Sanity field `images` (array), each with `_key`
- Special handling for `ctaSectionVideo`: upload video via `client.assets.upload('file', ...)` and use `fileRef()`
- For sub-objects (features array in featuresSection, links in contactInfo, etc.), add `_key: nanoid(12)` to each item
- Return null and log warning for unrecognized block types

**Singleton migrators (one function per singleton page):**
- `migrateHomePage()`: fetch `/api/home-page?populate=deep`, transform blocks, `createOrReplace({ _id: 'homePage', _type: 'homePage', title, description, blocks })`
- `migrateContactPage()`: fetch `/api/contact?populate=deep`, transform blocks, include `subTitle` field
- `migrateGalleryPage()`: fetch `/api/gallery?populate=deep`, transform blocks
- `migrateTrainingPage()`: fetch `/api/training?populate=deep`, transform blocks
- `migrateTrainingProgramsPage()`: fetch `/api/training-programs-page?populate=deep`, create document with `trainingPrograms` as array of `{ _type: 'reference', _ref: 'trainingProgram-${program.documentId}', _key: nanoid(12) }` — programs must be migrated first
- `migrateGlobalSettings()`: fetch `/api/global?populate=deep`, transform header and footer inline objects with their nested sub-objects (logo image, contact info, social links, hours, copyright link)
- `migrateMainMenu()`: fetch `/api/main-menu?populate[MainMenuItems][on][menu.dropdown][populate][sections][populate][links]=true&populate[MainMenuItems][on][menu.menu-link][populate]=true`, transform each item to either `menuDirectLink` (with `{ _type: 'menuDirectLink', _key, label, url }`) or `menuDropdown` (with `{ _type: 'menuDropdown', _key, label, sections: [{ _type: 'menuDropdownSection', _key, heading, links: [{ _type: 'menuLink', _key, label, url }] }] }`)

**Collection migrators:**
- `migrateTrainingPrograms()`: fetch all via `fetchStrapiPaginated('/api/training-programs')`, transform each to `{ _id: 'trainingProgram-${item.documentId}', _type: 'trainingProgram', title, description, slug: { _type: 'slug', current: item.slug }, blocks }`. Use transaction: `client.transaction()`, call `.createOrReplace(doc)` for each, then `transaction.commit({ visibility: 'deferred' })`
- `migrateContactForms()`: fetch all via `fetchStrapiPaginated('/api/contact-forms')`, transform each to `{ _id: 'contactForm-${item.documentId}', _type: 'contactForm', ...fields }`. Batch with transaction.
- `migrateGeneralInquiries()`: fetch all via `fetchStrapiPaginated('/api/general-inquiries')`, transform each to `{ _id: 'generalInquiry-${item.documentId}', _type: 'generalInquiry', ...fields }`. Batch with transaction.

**Summary report:**
- Track counts: `const counts: Record<string, { strapi: number, sanity: number }> = {}`
- After each content type migration, count Strapi items fetched and Sanity docs created
- After all migrations complete, query Sanity for each `_type` count: `*[_type == "${type}"] | count`
- Print summary table:
  ```
  [migrate] === Migration Summary ===
  [migrate] homePage: 1 Strapi -> 1 Sanity (OK)
  [migrate] trainingProgram: 5 Strapi -> 5 Sanity (OK)
  ...
  [migrate] Migration complete. Verify counts above before cleanup.
  ```

**Main execution order:**
```typescript
async function main() {
  console.log('[migrate] Starting Strapi -> Sanity migration')

  // Phase 1: Upload all images first to build imageCache
  await collectAndUploadAllImages()

  // Phase 2: Migrate collection types first (training programs needed for references)
  await migrateTrainingPrograms()
  await migrateContactForms()
  await migrateGeneralInquiries()

  // Phase 3: Migrate singleton pages
  await migrateHomePage()
  await migrateContactPage()
  await migrateGalleryPage()
  await migrateTrainingPage()
  await migrateTrainingProgramsPage()  // depends on training programs existing
  await migrateGlobalSettings()
  await migrateMainMenu()

  // Phase 4: Summary report
  await printSummaryReport()
}
```

The `collectAndUploadAllImages()` function should:
- Fetch all content types with `?populate=deep`
- Walk the response JSON recursively looking for objects with `url` property that match the Strapi media URL pattern (contains `/uploads/`)
- Upload each unique image URL and populate `imageCache`
- Log count: `[migrate] Found ${total} unique images to upload`

Each migrator function wraps its work in a try/catch. On error, log `[migrate] ERROR writing ${_type}: ${error.message}` and continue to next content type.
