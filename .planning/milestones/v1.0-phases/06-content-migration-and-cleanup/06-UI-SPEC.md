# Phase 6 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Phase Classification

**Type:** Infrastructure — no new user-facing UI surfaces.

This phase has two workstreams: (1) a one-shot migration script at `frontend/scripts/migrate-strapi-to-sanity.ts` that reads from the Strapi Cloud REST API, transforms all content, and writes to the production Sanity dataset; and (2) a cleanup pass that deletes every Strapi and Vercel Blob artifact from the codebase. No page components, layout components, or UI markup are added or changed. The site's rendered output is unchanged before and after.

The visual contract is a **preservation contract**: zero change to any user-visible output. End users experience the same UI they always have. The only observable change is that the Strapi backend and all references to it no longer exist in the repository.

## Typography

Source: `globals.css` `@theme inline` block. No changes in this phase — no new text elements are introduced.

**Constrained set (2 weights) for any new work in this phase:**

| Role | Size | Weight | Line Height | Font |
|------|------|--------|-------------|------|
| Body / Label / small | 14–18px (`--font-size-xs` – `--font-size-base`) | 400 (normal) | 1.5 | Montserrat |
| Heading / Display | 32–50px (`--font-size-3xl` – `--font-size-5xl`) | 600 (semibold) | 1.2 / 1.1 | Playfair Display / Cardo |

**Legacy note:** The existing codebase uses `font-bold` (700) and `font-extrabold` (800) in pre-existing component classes (`section-heading-red`, `btn-primary-red`, `card-heading`, etc.) and in layout component markup. These are preserved unchanged as part of the preservation contract. No new weights beyond 400 and 600 are introduced in this phase. Any new text elements added by the migration script are developer-facing terminal output only and carry no visual weight constraints.

## Interaction States

Phase 6 adds no interactive UI. Existing states in all layout components are unchanged.

### Migration Script States (terminal output only, never rendered)

| Condition | Behavior |
|-----------|----------|
| Script starts without required env vars | Throw with message listing missing vars; exit code 1 |
| Strapi API returns non-200 | Log error with content type and status; abort that content type; continue others |
| Image already exists in Sanity (deduplication hit) | Reuse existing asset reference; skip re-upload; log `[migrate] Image already exists: ${filename} — reusing` |
| Document already exists (`createOrReplace`) | Overwrite silently — idempotent by design |
| All content types complete | Print summary report with per-type counts and OK/MISMATCH status |
