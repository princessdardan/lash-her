7. `getMetaData(): Promise<TMetaData | null>` — Singleton fetch (globalSettings, just title/description).
   GROQ: `*[_type == "globalSettings"][0]{ title, description }`

8. `getTrainingProgramBySlug(slug: string): Promise<TTrainingProgram | null>` — Collection fetch with slug filter.
   GROQ: `*[_type == "trainingProgram" && slug.current == $slug][0]{ _id, title, description, "slug": slug.current, blocks[]{ _type, _key, heading, subHeading, description, onHomepage, image{ asset, hotspot, crop, alt }, link[]{ _key, href, label, isExternal }, info, name, email, phone, location, instagram, experience, interest, clients } }`
   Pass `{ slug }` as params to `client.fetch`.

9. `getAllTrainingPrograms(): Promise<TTrainingProgram[]>` — Collection fetch for all programs with full blocks.
   GROQ: `*[_type == "trainingProgram"]{ _id, title, description, "slug": slug.current, blocks[]{ _type, _key, heading, subHeading, description, onHomepage, image{ asset, hotspot, crop, alt }, link[]{ _key, href, label, isExternal }, info, name, email, phone, location, instagram, experience, interest, clients } }`

10. `getAllTrainingProgramSlugs(): Promise<Array<{ slug: string }>>` — Lightweight fetch for generateStaticParams.
    GROQ: `*[_type == "trainingProgram"]{ "slug": slug.current }`

**Export pattern (per D-01):**
```typescript
export const loaders = {
  getHomePageData,
  getContactPageData,
  getGalleryPageData,
  getTrainingsPageData,
  getGlobalData,
  getMainMenuData,
  getMetaData,
  getTrainingProgramBySlug,
  getAllTrainingPrograms,
  getAllTrainingProgramSlugs,
};
```

**Key GROQ patterns:**
- Singletons: `*[_type == "typeName"][0]{ ... }` — returns T | null
- Collections: `*[_type == "typeName"]{ ... }` — returns T[]
- Slug filter: `slug.current == $slug` with `client.fetch(query, { slug })`
- Slug projection: `"slug": slug.current` to unwrap Sanity slug object
- Image expansion: `image{ asset, hotspot, crop, alt }` — must explicitly project, not just `image`
- Portable text fields (perks, info, features on ctaFeature): project field name only (e.g., `perks`, `info`) — Sanity returns full Portable Text block array automatically
- All block projections include `_type, _key` as first two fields
- Each GROQ query is written inline using `groq` template tag

**Anti-patterns — DO NOT:**
- Import `qs`, `api`, `getStrapiURL`, or `TStrapiResponse`
- Use `qs.stringify()` for query building
- Wrap returns in `{ success, data, error }` envelope
- Use `...` spread without explicit field expansion for images
