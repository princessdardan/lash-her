**Step 1: Create frontend/src/components/ui/sanity-image.tsx (per D-04, D-05, D-06).**

New file. Do NOT modify strapi-image.tsx — it stays as-is per D-06.

```typescript
import Image from "next/image";
import imageUrlBuilder from "@sanity/image-url";
import { client } from "@/sanity/lib/client";
import type { TSanityImage } from "@/types";

const builder = imageUrlBuilder(client);

interface ISanityImageProps {
  image: TSanityImage;
  alt?: string;
  width?: number;
  height?: number;
  className?: string;
  fill?: boolean;
  priority?: boolean;
}

export function SanityImage({
  image,
  alt,
  className,
  priority,
  width,
  height,
  fill,
}: ISanityImageProps) {
  if (!image?.asset?._ref) return null;

  const url = builder.image(image).auto("format").fit("max").url();

  return (
    <Image
      src={url}
      alt={alt ?? image.alt ?? ""}
      className={className}
      priority={priority}
      {...(fill ? { fill } : { width: width ?? 800, height: height ?? 600 })}
    />
  );
}
```

Key implementation details:
- `builder` initialized at module level (not per-render) — avoids performance pitfall
- Accepts raw Sanity image object with asset._ref, hotspot, crop — per D-04
- The `@sanity/image-url` builder automatically reads hotspot and crop from the image object — per D-05 (no manual focalPoint call needed)
- Returns `null` when `image.asset._ref` is missing — same pattern as StrapiImage returning null for missing URLs
- Falls back to empty string alt for decorative images — matches UI-SPEC contract
- Uses `auto("format")` for automatic WebP/AVIF format selection — IMG-04
- Uses `fit("max")` to preserve aspect ratio
- Props: `fill` (for fill mode), or `width`/`height` (defaults 800x600) — mirrors StrapiImage prop pattern
